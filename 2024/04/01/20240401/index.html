Docker 建置打包 Python + Selenium + ChromeDriver + Chrome

發表於 2024-04-01 分類於 Learn ， Docker ， Python ， Selenium Disqus： 文章字數： 2.4k 所需閱讀時間 ≈ 6 分鐘

起因

使用 selenium 時，都必需要先抓對應的 ChromeDriver，
然後再開始使用，如果瀏覽器版本和 ChromeDriver 不同，會出現錯誤。

舉例

使用在下面 docker file build， google-chrome-stable 版本，
常常會發生跟抓的 Driver 對不上，程式進而會產生下列報錯等等

DockerFile

解決方法

Selenium 升級至 v4.0.0 以上 並搭配 webdriver_manager，
就可以不用特地去下載對應的 Driver 版本

webdriver_manager 會自動根據 dockerfile 中 google-chrome-stable版本，
自動在主機內安裝對應的 ChromeDrive 版本，並在 executable_path 這安裝取得路徑來做使用；
這樣以後使用 docker 打包就不會遇到版本的問題，也可以不用再下載 ChromeDrive，到專案中。
