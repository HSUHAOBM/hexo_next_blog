SOCKS5 Proxy 實作

發表於 2024-11-27 分類於 Learn ， Networking ， Proxy ， SSH ， VPS Disqus： 文章字數： 1.6k 所需閱讀時間 ≈ 4 分鐘

在先前實踐中使用過 SSH -R 實現內網穿透，將本地服務暴露到外網，這種方法有效解決了內網服務無法直接被訪問的問題。

這次，將嘗試使用用 SSH -D 配置 SOCKS5 Proxy，通過隧道轉發流量，不僅能隱藏 IP，還能輕鬆應對地區限制和匿名需求。

什麼是 Proxy？

Proxy（代理伺服器）是一種中介服務，位於用戶端與目標伺服器之間，負責轉發請求與回應。簡單來說，它是「中間人」，幫助用戶端隱藏自身資訊並與外界互動。

Proxy 的分類

Proxy 可根據用途與應用場景分為以下類型：

正向代理（Forward Proxy）

隱藏的是 用戶端的 IP，主要用於幫助用戶端訪問外部資源。常見用途包括：

  • 匿名瀏覽：避免暴露真實 IP，提高隱私性。
  • 突破地區限制：訪問受地區限制的網站或服務（如使用 VPN 或 SOCKS Proxy 瀏覽海外內容）。
  • 緩存加速：在企業內部使用 HTTP/HTTPS Proxy 提高網頁訪問效率。

反向代理（Reverse Proxy）

隱藏的是 伺服器的 IP，位於伺服器端，幫助內部伺服器處理外部請求。常見用途包括：

  • 保護伺服器安全：避免內部伺服器暴露在互聯網，降低攻擊風險。
  • 流量分配：將流量分發到多台伺服器以平衡負載（如 Nginx ）
  • 內容加速與緩存：透過 CDN 提供靜態資源加速和流量節省。

利用 SSH -D 實現 SOCKS5 Proxy

事前準備

  • VPS: 一台在雲端的虛擬私人伺服器，這邊使用 Linode，作為代理伺服器。
  • 本地機器：需要通過 SOCKS5 Proxy 繞過地區限制或隱藏真實 IP 的設備。

實際測試流程

1. 啟用 SOCKS5 Proxy

  • **-D 1080**：開啟本地端口 1080 作為 SOCKS5 Proxy 的入口。
  • **user@vps_ip**：連接的遠端伺服器帳號與 IP。
  • 加密隧道：數據流量經由 SSH 隧道傳輸，確保隱私和安全。
    執行指令後，確認 SSH 成功連接且本地代理埠（如 1080）正常運行。

2. 配置本地機器使用代理

Firefox 瀏覽器

  1. 進入「設定 > 一般 > 網路設定 > 連線設定」。
  2. 選擇「手動設定 Proxy」：
    • SOCKS 主機：127.0.0.1
    • 埠號：1080
    • 協議：選擇 SOCKS5。
  3. 保存設置後測試連線。
  4. 查看 Ip

補充

Netflix 測試

使用 ssh -D 成功設置了 SOCKS5 Proxy，並測試 Netflix 的訪問。結果顯示：

  • 代理效果：代理生效且成功切換至日本版內容，說明 SOCKS5 Proxy 隧道正常運作。
  • 登入受限：在嘗試登入帳號時，多次遭到阻擋，推測可能原因為 Netflix 偵測到代理行為或伺服器 IP 被標記為非正常流量來源。
    該方法適合短期代理需求，但對 Netflix 等流媒體應用仍有限制。

SOCKS5 Proxy 與 VPN 的對比

特性 SOCKS5 Proxy（SSH -D） VPN
加密範圍 代理應用層流量（特定應用程式，例如瀏覽器）。 全系統流量（包括 DNS 和所有網路活動）。
代理層級 應用層（僅代理特定應用程式的流量）。 網路層（包括所有網路流量和系統應用）。
適用場景 短期臨時使用、匿名瀏覽、特定流量的代理需求。 長期穩定使用、影音流媒體、全面隱私保護。
被偵測風險 高，容易被流媒體服務檢測到。 低，通常能繞過流媒體檢測。
配置難度 簡單，僅需設置應用程式代理端口。 需要安裝專用軟體並配置 VPN 憑證。
用途 適合短期匿名需求，例如遊戲、影音、網頁代理。 適合長期使用，提供全面安全和隱私保護。
